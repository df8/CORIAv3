CORIA is a system designed to analyse networks (or graphs) of any size. In the context of Autonomous Systems (AS) we consider the Internet a network of interconnected routers where an AS can be represented by a node and the physical link between any two AS as an edge. This allows us to transfer research problems such as measuring global and regional Internet resilience into known and in some cases already solved mathematical problems within the field of graph theory.

The CAIDA import module supports the following file type. You can drag and drop files in any order, as CORIA accepts multiple files with the described format and merges all provided information into a single list of AS Organizations.

Autonomous System Organizations Dataset

At least one file of this type is required.

In their effort to infer which organizations operate which Autonomous System(s), since 2009 CAIDA quarterly scans WHOIS database dumps and extracts the organization listed for each AS.[1][2] Understanding how various parts of the Internet infrastructure are distributed among the operating organizations allows to unravel "the topological and economic structure of the Internet"[2]. An example of a WHOIS entry can be found under [3].

This import module allows you to import a batch of organizations from files provided by CAIDA.

Format definition for AS Organization files: [4]

organization ID|date of last entry change|organization name|country ID|source database
Example: 
HARVAR-ARIN|20190812|Harvard University|US|ARIN

Interpretation: From the above line, we add a new entry for the organization Harvard University, initially found via a WHOIS lookup in the database ARIN, located in US and identified by HARVAR-ARIN.

Download: Compatible AS organization files can be obtained from data.caida.org: e.g. 20200401.as-org2info.txt.gz Link: http://data.caida.org/datasets/as-organizations/

Please extract the text file from the caida.org-provided .gz archive before uploading.
