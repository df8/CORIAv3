CORIA is a system designed to analyse networks (or graphs) of any size. In the context of Autonomous Systems (AS) we consider the Internet a network of interconnected routers where an AS can be represented by a node and the physical link between any two AS as an edge. This allows us to transfer research problems such as measuring global and regional Internet resilience into known and in some cases already solved mathematical problems within the field of graph theory.

The CAIDA import module supports the following file type. You can drag and drop files in any order, as CORIA accepts multiple files with the described format and merges all provided information into a single list of AS Locations.

Autonomous System Locations Dataset

At least one file of this type is required.

In order to understand technical and economic implications for the Internet stability, CAIDA publishes an association mapping between pairs of AS (AS links) and their estimated geographic locations.[1] "Understanding the geographic nature of these relationships can facilitate activities such as: realistic simulation of AS path prediction; application performance estimation; predicting the likelihood that two ASes will interconnect; and visualizing the geographic distribution of networks."[1] This import module allows you to import a batch of locations from files provided by CAIDA.

Format definition for AS Location files: [2]

location ID|continent ID|country ID|region ID|city|latitude|longitude|population number
Example: 
Berlin-BE-DE|EU|DE|BE|Berlin|52.5303|13.3333|0

Interpretation: From the above line, we add a new entry for the city Berlin in the region BE (state of Berlin) of DE (Germany) located in EU (Europe) and having the GPS coordinates 52.5303 N, 13.3333 E.

Download: Compatible AS location files can be obtained from data.caida.org: e.g. 201603.locations.txt.gz Link: http://data.caida.org/datasets/as-relationships-geo/

Please extract the text file from the caida.org-provided .gz archive before uploading.
